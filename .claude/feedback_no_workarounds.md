---
name: No workarounds — fix root cause
description: Never bypass a problem with an alternative approach; always fix the actual issue
type: feedback
---

When something fails (wrong token scope, missing tool, wrong flag), identify and fix the root cause. Do not try alternative approaches to work around it.

**Why:** User explicitly stated this. Trial-and-error and workarounds waste time and obscure what is actually happening.

**How to apply:** If a command fails due to a missing permission, wrong version, or misconfiguration — stop, explain the root cause clearly, and fix it directly. Ask the user to take corrective action if needed (e.g. update a token scope) rather than switching to a different tool or API.
